# Quickstart: 小説モード追加の検証フロー

**Feature**: `002-add-novel-mode` | **Date**: 2026-06-13

各ユーザーストーリーを独立に検証する手順。`npm run dev:frontend`（＋ AI を動かす場合 `npm run dev:functions`）でローカル起動して実施する。各ストーリーは単独で「動く MVP スライス」として検証可能。

## 前提

```bash
npm install
npm run dev:functions   # 別シェル。AI/応募検証に必要
npm run dev:frontend    # http://localhost:5173
```

ログイン（Google Sign-In）済みのテストユーザーで実施。

---

## US1（P1）: 小説作品を新規作成し章立てで執筆

1. カタログで「新規作成」→ モード選択モーダルで **「小説」** を選ぶ。
2. 新規ドキュメントが `contentType: 'novel'` で作成され、**章ツールバー＋章一覧パネル**が表示される。
3. ツールバーで「新しい章」「新しい節」「会話」「地の文」を挿入 → 章一覧パネルに反映される。
4. 保存 → ページ再読み込み → 全章・節・本文・章タイトルが一致。
5. （互換）既存脚本を開く → 脚本モードのエディタ（柱・ト書き・セリフ）で開く。

**Pass 条件**: 4 の往復で構造が完全復元。5 で脚本挙動が変わらない。

---

## US2（P2）: 小説向け AI アドバイス 2 パネル

1. 小説ドキュメントにあらすじ＋本文を入力。
2. アドバイス要求 → **パネルA=編集者 / パネルB=文芸評論家**で応答。
3. 応答プロンプトに「柱」「ト書き」「ペラ」が含まれないことを確認（SC-007）。
4. 同ユーザーの脚本ドキュメントでアドバイス → 脚本プロンプトが使われ、結果が `contentType` ごとに分離保存。

**Pass 条件**: モードごとに正しいプロンプト。混線語ゼロ。応答 < 10 秒（SC-003）。

---

## US3（P2）: 小説向け原稿エクスポート

1. 章・節・本文入りの小説ドキュメントで docx エクスポート。
2. 章=「見出し1」、節=「見出し2」、本文=段落字下げで出力。書字方向は設定どおり（既定縦書き）。
3. プリセット「縦書き原稿用紙」「文庫横書き」「Web小説プレーン」を切替 → 体裁が変わる。
4. （互換）既存脚本をエクスポート → 従来の縦書き脚本グリッド／登場人物→あらすじ→本文順。

**Pass 条件**: Word/Pages/LibreOffice で章節階層が正しく表示（SC-004 ≥95%）。脚本エクスポート不変。

---

## US4（P3）: モード切替 UI とカタログフィルタ

1. 脚本 1 件・小説 1 件を持つ状態でカタログを開く → 上部に「すべて/脚本/小説」、既定「すべて」。
2. 各カードに **モードバッジ**（脚本/小説）。
3. 「小説」タブ → 小説のみ表示、件数連動。
4. 「新規作成」→ モード選択モーダル → 選んだモードで作成。
5. ナビ・見出し・empty state は「マイ作品」総称（Q5）。

**Pass 条件**: フィルタで目的の作品に 3 クリック以下で到達（SC-006）。

---

## US5（P4）: コンテスト・グループに小説提出

1. 管理者がコンテスト作成 → 「対応モード」で `screenplayOnly`/`novelOnly`/`both` を選択。
2. 「小説のみ」コンテストの応募画面 → 自分の小説のみが候補（脚本は非表示）。
3. （防御）API 直叩きで脚本を「小説のみ」へ応募 → **403**（`CONTENT_TYPE_NOT_ALLOWED`）。
4. 講評者が小説 Submission を開く → 章一覧＋小説プロンプト AI アドバイス。
5. グループ作品一覧でモードバッジ＋フィルタ。
6. （互換）既存脚本コンテスト／グループの応募・閲覧・添削がリグレッションなく動作。

**Pass 条件**: 小説応募成功率 ≥95%（SC-008）。既存脚本フロー 30 日ゼロ件（SC-009）。

## マイグレーション検証

```bash
# dry-run（書込なし、件数確認）
npx tsx scripts/migrate-content-type.ts --dry-run
# 本実行（バックアップ取得後）
npx tsx scripts/migrate-content-type.ts
```

**Pass 条件**: 実行後 `contentType` / `supportedModes` 未設定件数 0。既存脚本に影響なし。
